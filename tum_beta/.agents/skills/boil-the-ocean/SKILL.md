# Protocolo "Boil the Ocean"

Este protocolo define el estándar de excelencia para cada intervención en este workspace. No se aceptan soluciones parciales ni planes de ejecución futuros cuando el producto terminado puede ser entregado ahora.

## 1. Principios de Ejecución
- **Costo Marginal Cero**: La completitud no es opcional. Si una tarea requiere documentación, pruebas y refactorización, se hace todo en un solo ciclo.
- **Efecto "Holy Shit"**: El objetivo no es cumplir, sino impresionar. La solución debe ser tan robusta y pulida que no deje lugar a dudas.
- **Cierre Total**: No se dejan hilos sueltos. Si amarrar un detalle toma 5 minutos más, se hace.
- **Arreglos Permanentes**: Prohibido proponer "workarounds" o "lo dejamos para luego" si existe una solución definitiva.

## 2. Flujo de Trabajo Obligatorio
1. **Investigación Profunda**: Buscar antes de construir. Entender el contexto completo del sistema.
2. **Implementación Integral**: Escribir el código, pero también la documentación y las pruebas unitarias/integración necesarias.
3. **Validación Rigurosa**: Probar exhaustivamente antes de dar por terminada la tarea.
4. **Entrega de Producto**: No presents un plan de acción. Presenta el producto funcionando y verificado.

## 3. Sin Excusas
- La complejidad técnica es un reto, no una limitación.
- El tiempo no es factor para sacrificar la calidad.
- La fatiga no existe en la búsqueda de la perfección del código.

## 4. Checklist de Salida
- [ ] ¿Es la solución permanente?
- [ ] ¿Está documentada adecuadamente?
- [ ] ¿Tiene pruebas que validen su funcionamiento?
- [ ] ¿He amarrado todos los hilos sueltos?
- [ ] ¿El resultado genera un impacto de "Producto Terminado"?

> "El estándar no es 'con que alcance, ya está' — es 'holy shit, ya quedó hecho'."
